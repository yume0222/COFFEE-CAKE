# カフェサイト

## 制作背景
複数のレイアウト（シングルカラム、2カラム、タイル型、ブロークングリッド）を用いたWebサイト制作を通して、コンテンツに応じた適切な情報設計とレイアウト表現を身につけることを目的に制作しました。<br>
また、グリッドレイアウトを活用し、要素の整列や余白のバランスを意識した、視認性の高い画面設計に取り組みました。<br>
ブロークングリッドの実装では、意図した位置に要素を配置することが難しく、レイアウトが崩れてしまうこともありました。<br>
さらに、GoogleマップやYouTubeの埋め込みを実装することで、外部コンテンツを取り入れた実用的なWebサイト構築の理解を深めました。

---

## 使用技術
HTML, CSS, JavaScript

---

## 工夫した点
CSS設計を意識し、BEMやコンポーネント化、CSS変数の活用を行いました。<br><br>

制作後もアクセシビリティや保守性について学習を続け、新たに得た知識をもとに既存のコードを見直しました。<br>
WAI-ARIAを活用したアクセシブルなハンバーガーメニューを実装しました。aria-expandedやaria-controlsなどをJavaScriptと連動させ、スクリーンリーダー利用者にも開閉状態が伝わるよう配慮しています。<br>
コメントアウトを追加・整理し、可読性と保守性を向上させました。

---

## 学んだこと
ブロークングリッドの実装を通して、見た目を再現するためにはCSSの仕様を正しく理解し、複数のプロパティを組み合わせて調整することの重要性を学びました。<br>
また、ハンバーガーメニューを自分で構築する中で、動作の仕組みやコードの意味を一つひとつ理解しながら実装する力が身につきました。<br>
分からない点は調べたりAIを活用したりしながら解決することで、自力で実装できる範囲が広がったと感じています。
